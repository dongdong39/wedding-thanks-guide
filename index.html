GitHub 회원가입

GitHub는 전 세계 수천만 개발자들이 사용하는 온라인 코드 저장소입니다. 코드를 저장하고 공유할 수 있으며, 웹페이지를 무료로 호스팅할 수도 있습니다.

  1. https://github.com/signup에 접속합니다.
  2. 이메일, 비밀번호, 사용자 이름(username)을 입력합니다.
  3. 이메일 인증을 완료하면 가입 끝!
사용자 이름(username)이 중요합니다!
나중에 감사카드 주소가 https://사용자이름.github.io/레포이름/ 형태가 됩니다.
예: 사용자이름이 honggildong이면 → https://honggildong.github.io/wedding-thanks/
💡 이미 GitHub 계정이 있다면 이 단계를 건너뛰세요!

감사 인사말 수정하기

index.html 파일을 메모장이나 텍스트 편집기로 열어 수정합니다.

🍎 맥 텍스트 편집기 — HTML 코드로 보기 (코드로 안 보이는 경우 클릭)

맥의 기본 텍스트 편집기는 HTML 파일을 열면 코드가 아니라 렌더링된 웹페이지처럼 보여줍니다. 아래 설정을 바꾸면 코드로 볼 수 있어요.

  1. 텍스트 편집기 실행 → 메뉴 막대에서 텍스트 편집기 → 설정
  2. 열기 및 저장하기 탭 클릭
  3. "HTML 파일을 포맷 텍스트 대신 HTML 코드로 표시" 체크
  4. 파일을 다시 열면 코드가 보입니다
💡 윈도우 메모장은 별도 설정 없이 바로 코드로 열립니다.

GitHub에 올리기

  1. github.com/new에서 새 레포지토리를 만듭니다.
  2. Repository name에 wedding-thanks를 입력합니다.
  3. Public을 선택합니다. (무료 배포를 위해 필수!)
  4. Create repository 버튼을 클릭합니다.
  5. 생성된 페이지에서 "uploading an existing file" 링크를 클릭합니다.
  6. 다운받은 파일들을 모두 드래그 앤 드롭합니다.
⚠️ images 폴더째로 드래그하세요! 파일을 하나씩 넣으면 폴더 구조가 깨집니다.
💡 업로드 후 하단의 Commit changes 버튼을 꼭 클릭하세요!
7

웹페이지 배포하기 (GitHub Pages)

  1. 레포지토리 페이지에서 상단 Settings 탭을 클릭합니다.
  2. 왼쪽 메뉴에서 Pages를 클릭합니다.
  3. Source 항목에서 Deploy from a branch를 선택합니다.
  4. Branch를 main으로 선택하고 Save를 클릭합니다.
  5. 1~2분 후 상단에 초록색 링크가 나타납니다!
✅ 링크 형식: https://사용자이름.github.io/wedding-thanks/

자주 묻는 질문

📸 사진 관련

index.html에서 갤러리 부분을 찾아 <div class="gallery-item">...</div> 줄을 추가하거나 삭제하세요. 4의 배수로 맞추면 깔끔합니다. 파일 이름은 g17.jpg, g18.jpg 처럼 순번을 이어가면 됩니다.

index.html에서 <div class="gallery-item"> 줄의 순서를 바꿔주면 됩니다. 또는 파일 이름 번호를 서로 바꿔도 됩니다. (예: g1.jpg와 g3.jpg를 서로 교환)

파일 이름을 확인해주세요.
- 대소문자가 정확한지? (G1.jpg ✖ → g1.jpg ✔)
- 확장자가 .jpg인지? (.jpeg, .png, .JPG는 안 됩니다)
- images 폴더 안에 있는지?

HEIC 파일(아이폰 기본 형식)은 JPG로 변환해야 합니다. iLoveIMG에서 무료 변환 가능합니다.

사진 1장당 500KB~1MB 이하를 추천합니다. iLoveIMG 압축에서 무료로 용량을 줄일 수 있습니다.

🎬 영상 관련

GitHub은 파일당 25MB 이하만 업로드 가능합니다. 영상 압축 방법:
- FreeConvert에서 무료 압축
- 아이폰: 설정 → 카메라 → 해상도 낮춰서 다시 내보내기
- 5MB 이하로 맞추는 것을 추천합니다.

index.html에서 <details class="ceremony-item"> 블록을 통째로 삭제하면 해당 항목이 사라집니다. <summary> 안의 텍스트를 바꾸면 제목이 변경되고, <source src="...">의 파일 이름을 바꾸면 다른 영상으로 교체됩니다.

index.html에서 <!-- 식순 영상 -->부터 해당 </section>까지 통째로 삭제하면 됩니다.

파일 형식이 .mp4인지 확인해주세요. .mov, .avi 등은 일부 브라우저에서 재생이 안 됩니다. FreeConvert에서 MP4로 변환할 수 있습니다.

🔧 배포 & 공유 관련

GitHub Pages 반영에 1~2분 정도 걸립니다. 그래도 안 보이면 브라우저 캐시 때문입니다.
- PC: Ctrl + Shift + R (Mac: Cmd + Shift + R)
- 모바일: 시크릿/프라이빗 모드로 열기
- 카카오톡 인앱 브라우저: 링크 길게 눌러 "외부 브라우저로 열기"

카카오톡은 한번 읽어간 URL의 정보를 캐시합니다.
카카오 공유 디버거에서 URL을 입력하고 캐시 초기화를 누르면 새 정보로 갱신됩니다.

미리보기 제목/설명을 바꾸려면 index.html 상단의 <meta property="og:title">과 <meta property="og:description">을 수정하세요.

index.html 상단의 <meta property="og:image" content="images/g1.jpg"> 부분에서 원하는 이미지 파일명으로 변경하세요. 변경 후 카카오 공유 디버거에서 캐시를 초기화해야 반영됩니다.

💰 비용 & 기간 관련

네, 완전 무료입니다. GitHub Pages는 Public 레포지토리에 대해 무료로 웹 호스팅을 제공합니다. 도메인 비용도 없고, 트래픽 비용도 없습니다. 추가 결제를 요구하지 않습니다.

기간 제한이 없습니다. 직접 삭제하지 않는 한 영구적으로 유지됩니다. GitHub 계정을 유지하기만 하면 됩니다.

GitHub에서 해당 레포지토리를 삭제하면 페이지도 함께 사라집니다.
레포지토리 → Settings → 맨 아래 Danger Zone → Delete this repository

GitHub Pages는 월 100GB 대역폭을 제공합니다. 일반적인 감사카드 페이지는 수천 명이 방문해도 충분합니다. 걱정하지 않으셔도 됩니다.

📝 내용 수정 관련

HTML 태그가 깨졌을 가능성이 높습니다. 흔한 실수:
- <p>는 있는데 </p>가 빠진 경우
- <br>을 <br/>이나 다른 형태로 잘못 쓴 경우
- 따옴표(")를 실수로 지운 경우

원본 파일과 비교하면서 확인해보세요. 최악의 경우 원본 템플릿을 다시 다운받아서 이름만 다시 수정하면 됩니다.

parents.html 파일을 업로드하지 않으면 됩니다. 또는 업로드했더라도 링크를 공유하지 않으면 아무도 볼 수 없습니다.

styles.css 파일에서 background: url('images/g1.jpg') 부분을 원하는 이미지 파일명으로 변경하세요. 가로로 넓은 사진이 잘 어울립니다.
